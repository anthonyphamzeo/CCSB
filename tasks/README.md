# Task Tracking

The folder containing a task record represents its current state.

- `backlog/`: approved or proposed work not yet started.
- `in-progress/`: actively owned tasks with a branch and acceptance plan.
- `completed/`: accepted tasks with implementation and validation evidence.
- `releases/`: ordered, release-specific scope and readiness lists.
- `templates/`: standard task and release records.

Use one Markdown file per task: `US-###-T##-short-description.md`. Move the file rather than copying it when state changes.
