# Documentation

Documentation is versioned with the implementation it describes.

| Area | Content |
|---|---|
| `tracking/` | Delivery status, decisions, risks/issues, dependencies, and completed-work evidence |
| `design/` | Architecture, ADRs, data model, API/integration, security, and UX artefacts |
| `implementation/` | Build records, configuration, deployment, test evidence, operations, and release notes |

Large generated reports should be published as pipeline artefacts. Commit concise source documents and durable evidence indexes here.
