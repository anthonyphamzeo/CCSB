# Tracking Documents

- `status/`: concise project, milestone, and workstream status.
- `risks-issues/`: owned risks, issues, mitigations, and target dates.
- `decisions/`: delivery and governance decisions that do not belong in architecture ADRs.
- `dependencies/`: cross-team, platform, and release dependencies.
- `completed-work/`: durable index of accepted tasks and their evidence.

Task state itself is maintained in `tasks/`; this area holds reporting and evidence, not duplicate task truth.
