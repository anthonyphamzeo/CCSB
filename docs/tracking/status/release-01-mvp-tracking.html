Release Interpretation

The updated backlog adds the Configuration Studio PCF. For this first release, implement US-014-T01 and US-014-T02 so the boundary, information architecture and deployable shell exist. Complete US-014-T03 through US-014-T11 after the Runtime PCF and operation contracts prove the configuration shape.

Status, owner, sprint and notes autosave to this browser. Use Save JSON DB to persist a portable file database and Load JSON DB to restore it later or move it to another machine.

ALM Sequencing Rules

  • Keep schema, Custom APIs, plug-ins, PCFs, security roles and configuration assets in the solution/repository from the start.
  • Export or back up the unmanaged development solution before schema changes while only one environment exists.
  • Preserve the no-direct-write rule: PCFs read governed slices and mutate only through versioned Custom APIs.
  • Use one work item branch per task and keep schema, API and PCF concerns reviewable.
  • Start accessibility and instrumentation early; reserve formal performance, upgrade and assistive-technology evidence for hardening.
