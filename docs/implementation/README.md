# Implementation Records

| Folder | Purpose |
|---|---|
| `build-records/` | Important implementation notes and generated-component provenance |
| `configuration/` | Environment variable, feature flag, and connection reference guidance |
| `deployment/` | Install, upgrade, rollback, and clean-environment deployment records |
| `testing/` | Test plans, validation reports, and release evidence |
| `operations/` | Monitoring, alerting, incident, support, backup, and recovery runbooks |
| `release-notes/` | Versioned release notes and known limitations |

Never store credentials, raw production data, or sensitive telemetry in implementation records.
